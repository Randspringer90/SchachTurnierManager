# License decision required

Status: **OWNER_DECISION_REQUIRED**

GitHub currently reports this repository as public with no detected license, and no root
`LICENSE`/`COPYING` file exists. The official Build Week rules require a public judging repository
to have relevant licensing, or permit a private repository shared with the specified judging
accounts.

No license is selected in this run. Adding one silently could relicense code contributed by
others without confirming that the Owner has the necessary rights and agreement.

## Option A — public repository with an explicit open-source license

Before choosing a license, the Owner should:

1. inventory original and contributed code, including adapted work from Marcel;
2. confirm contribution terms or obtain explicit contributor agreement where needed;
3. select a license compatible with intended distribution and all included third-party material;
4. add the exact license text, copyright/attribution notices, and any required source/notice files;
5. rerun repository, package, installer, Android, and submission checks.

Potential permissive or copyleft licenses can be evaluated, but this document intentionally does
not recommend one without the rights and product-distribution decision.

## Option B — private repository with jury access

The official rules allow a private repository if it is shared with the two judging addresses
listed on the challenge page. This avoids implying an open-source grant, but it does not remove
third-party compliance, ownership, privacy, testing-access, or submission-period evidence duties.
Changing visibility and granting access are Owner-only external actions and are not performed here.
